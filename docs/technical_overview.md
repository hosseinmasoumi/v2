# مستند فنی سامانه پایش کیفیت پروژه‌های راه‌سازی

این سند برای ارائه فنی پروژه آماده شده و تلاش می‌کند به صورت خلاصه ولی کاربردی به مهم‌ترین سؤال‌هایی که ممکن است در جلسه مطرح شود پاسخ بدهد.

---

## ۱. پشتهٔ فناوری (Tech Stack)

| لایه | فناوری / کتابخانه | توضیح |
|------|-------------------|-------|
| Backend | **Django 5** (Python) | فریم‌ورک اصلی برای پیاده‌سازی API، لایه مدیریتی و منطق تجاری. |
| ORM | Django ORM | مدیریت مدل‌های داده و مهاجرت‌ها. |
| پایگاه داده | SQLite (محلی) | برای توسعه استفاده شده؛ در استقرار می‌تواند به PostgreSQL یا MySQL تغییر کند. |
| قالب‌ها | Django Template، Jinja-like syntax | برای ساخت صفحات HTML، و استفاده از contextهای دینامیک. |
| Frontend | HTML5، Bootstrap 5، Select2، jQuery | برای UI/UX، فرم‌های پویا و تعاملات AJAX. |
| داشبورد | Canvas API سفارشی، فایل‌های JS ماژولار (`static/js/dashboard.js` و ... ) | رندر پروفیل طولی، لایه‌ها، ساختارها و بازه‌ها به صورت تعاملی. |
| تقویم شمسی | `persian-date` / `persian-datepicker` | برای تاریخ‌های فارسی در فرم‌ها. |
| اسکریپت‌های مدیریت | Django Management Commands | برای seed کردن داده‌ها، افزودن انواع آزمایش و ... |

---

## ۲. معماری و سازمان کد

1. **معماری مبتنی بر اپلیکیشن**: اپ‌های جداگانه برای حوزه‌ها (`project`، `experiment`، `core`).  
2. **جداسازی منطق از نمایش**:
   - مدل‌ها در فایل‌های `models.py` هر اپ.  
   - Formها در `forms.py` برای اعتبارسنجی و سفارشی‌سازی ویجت‌ها.  
   - Viewها هم از کلاس‌های جنریک Django (CRUD) و هم از فانکشن‌ویو (برای AJAX) استفاده می‌کنند.  
   - Templateها در پوشهٔ `templates/<app>/...` و از inheritance برای header/footer و ... استفاده می‌شود.
3. **لایهٔ فرانت سفارشی برای داشبورد**:
   - ماژول‌های JS مجزا (`dashboard.js`, `canvas.js`, `xaxiscanvas.js`, `yaxiscanvas.js`) برای مدیریت کانواس و رندر.  
   - داده‌ها با JSON از view به template تزریق شده و در JS مصرف می‌شوند.
4. **Commandهای مدیریت** برای دادهٔ اولیه:
   - `experiment/management/commands/add_experiment_data.py`: پر کردن انواع آزمایش و زیرنوع‌ها.  
   - `project/management/commands/seed_projects.py`: ایجاد پروژه‌های نمونه با لایه‌های مختلف، ابنیه و کاربر پیش‌فرض.

---

## ۳. داشبورد پایش پروژه

### داده‌های ورودی
- فایل پروفیل پروژه (Excel) برای ترسیم پروفیل طولی.  
- مدل‌های `ProjectLayer` و `ProjectStructure` برای لایه‌ها و ابنیه.  
- درخواست‌ها و نتایج آزمایش (`ExperimentRequest`, `ExperimentResponse`) برای shading و بازه‌های اجرا شده.

### جریان رندر
1. **Backend**: `ProjectDashboardView` دادهٔ پروژه را جمع‌آوری کرده و به صورت JSON در قالب تزریق می‌کند (`project/templates/project/dashboard.html`).  
2. **Frontend**:  
   - `dashboard.js` Canvas اصلی را می‌سازد، مقیاس‌ها را محاسبه می‌کند، لایه‌ها و بازه‌ها را رسم می‌کند.  
   - برای مقادیر کیلومتر/ارتفاع از `transformX` و `transformY` استفاده می‌شود.  
   - Tooltip ها، شماره‌گذاری لایه‌های متغیر، بازه‌های کیلومتراژ و ساختارهای پروژه (پل، آبرو، …) همگی در همین فایل مدیریت می‌شود.  
3. **ویژگی‌های کلیدی**:
   - شماره‌گذاری خودکار لایه‌های تکراری (مثلاً «آسفالت گرم ۱»، «آسفالت گرم ۲») در هم داشبورد و هم فرم‌ها.  
   - بخش‌بندی shading بین پروفیل زمین و جاده فقط در نواحی زیر محور صفر بر اساس تحلیل متر به متر.  
   - فرم‌های فیلتر، تولتیپ دقیق و مدیریت بازه‌های کیلومتری با FormSetهای پویا.

---

## ۴. مدیریت پروژه‌ها و لایه‌ها

### مدل‌های کلیدی
- `Project`: مشخصات کلی پروژه (مسافت، مدیران، کیلومتر شروع/پایان، فایل پروفیل و …).  
- `ProjectLayer`: نوع لایه، ضخامت، ترتیب از بالا، حالت (ثابت/متغیر) و وضعیت اجرا.  
- `ProjectStructure`: ابنیه‌ی مرتبط با پروژه (پل، آبرو، تونل و …).

### امکانات مهم
1. **صفحه «لیست لایه‌ها»** (`project-layer-list`):
   - نمایش خلاصه از تعداد استفاده هر نوع لایه + ترتیب‌ها.  
   - کارت‌های مجزا با شماره‌گذاری «نام ۱»، «نام ۲» برای لایه‌های هم‌نوع.  
2. **فرم «ایجاد/ویرایش لایه»**:  
   - امکان کپی از لایه‌های قبلی و شماره‌گذاری خودکار.  
   - اعتبارسنجی تکرار ترتیب.
3. **فرم «درخواست آزمایش»** (`experiment` اپ):
   - انتخاب پروژه → بارگذاری AJAX لایه‌ها با شماره‌گذاری.  
   - فرم‌ست بازه‌های کیلومتراژ + توضیح بازه و فایل‌های پیوست.  
   - ثبت بازه‌های متعدد، تولید خودکار start/end کیلومتر از روی اولین/آخرین بازه.

---

## ۵. سمت کاربر و فرانت‌اند

- **Bootstrap 5** برای استایل و layout.  
- **Select2** روی فیلدهای انتخابی برای UX بهتر و انتخاب چندگانه.  
- **jQuery** برای هندل کردن رویدادهای فرم، افزودن ردیف FormSet، و AJAX.  
- **Persian Date Picker** برای تاریخ‌های شمسی.  
- **CSS اختصاصی** برای کارت‌ها، badge‌ها و افکت‌های hover.

---

## ۶. پنل ادمین و ابزارهای مدیریت

- از **Django Admin** برای مدیریت سریع مدل‌ها استفاده شده.  
- مدل‌های اضافی (`ProjectLayer`, `ProjectStructure`, `ExperimentRequest`, ...) در admin ثبت شده‌اند.  
- در صورت نیاز می‌توان از `django-grappelli` یا تم‌های مشابه برای ظاهر بهتر استفاده کرد (پیش‌نیاز آن نصب پکیج است؛ در محیط فعلی بسته نصب نشده و به همین دلیل اجرای مستقیم `manage.py shell` ارور عدم وجود این پکیج را داد).

---

## ۷. دادهٔ نمونه و Seed

- اجرای `python manage.py seed_projects` چند پروژهٔ نمونه با ساختار واقعی ایجاد می‌کند (لایه‌های تکراری، ابنیه‌ها و مدیران).  
- اجرای `python manage.py add_experiment_data` انواع آزمایش و زیرنوع‌ها را پر می‌کند.  
- این دستورات برای آماده‌سازی سریع دمو و تست در محیط توسعه استفاده می‌شوند.

---

## ۸. تست و اعتبارسنجی

- در حال حاضر تمرکز روی تست‌های دستی و بررسی UI/UX بوده است.  
- مسیرهای حیاتی مثل ثبت لایه، ثبت درخواست آزمایش، مدیریت بازه‌ها و رندر داشبورد به صورت دستی تست شده‌اند (شامل حالت‌هایی مثل لایه‌های تکراری، بازه‌های متعدد، حذف بازه و …).  
- برای مرحله بعدی می‌توان از test caseهای Django برای مدل‌ها و APIها استفاده کرد.

---

### نکات قابل اشاره در جلسه

- ساختار ماژولار پروژه (app به app) + جداسازی منطق (views/forms) امکان توسعه و نگهداری راحت را فراهم کرده.  
- استفاده از Canvas سفارشی باعث شده نیاز به کتابخانه‌های سنگین charting نداشته باشیم و رفتار دقیق‌تری روی shading و بازه‌های منفی داشته باشیم.  
- شماره‌گذاری خودکار لایه‌ها و بازه‌ها در تمام بخش‌ها هماهنگ شده و از بروز اختلاف بین داشبورد، فرم‌ها و گزارش‌ها جلوگیری می‌کند.  
- وجود فرمان‌های seed باعث می‌شود هر توسعه‌دهنده‌ای بتواند به سرعت محیط دمو را آماده کند.

---

> اگر در جلسه پرسشی دربارهٔ «این صفحه با چی پیاده شده؟» یا «چطور دادهٔ لایه‌ها شماره خورده؟» مطرح شد، می‌توانید به بخش‌های بالا ارجاع بدهید. در اکثر موارد، دانستن این‌که Backend روی Django است و فرانت داخل قالب‌ها با JS/Canvas شخصی نوشته شده، پاسخ را کامل می‌کند.
